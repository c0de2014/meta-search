# JSURLs (NoAddURL,NoAppend)
#
# Die Liste "JSURLs" wird benutzt, um externe Skripten auch dann korrekt behandeln
# zu können, wenn sie vom Server mit einer falschen Content-Type-Angabe
# übertragen werden.
#
# Der Seitenfilter "Prox: URLs externer Skripten speichern" durchsucht Webseiten
# nach Verweisen auf externe Skripten und trägt gefundene URLs in die Liste
# "JSURLs" ein. Entsprechende Headerfilter sorgen dann dafür, dass
# Content-Type-Angaben dieser Skripten, wenn nötig, korrigiert werden.
#
# Damit sich nicht mit der Zeit riesige Datenmengen in der Datei ansammeln, werden
# die während der Laufzeit gesammelten Einträge nicht in die Datei geschrieben,
# sondern nur im Speicher als Teil der Liste behandelt.
# Sobald Proxomitron beendet oder die Listen neu eingelesen werden, werden die
# temporären Listeneinträge gelöscht.
# Man kann sich "JSURLs" also als eine Mischung aus statischer und virtueller Liste
# vorstellen. Als Hybrid-Liste, so zu sagen.
#
# Die in dieser Datei stehenden statischen Listeneinträge sollen besonders häufig
# vorkommende URLs abdecken, um das doch recht komplexe Korrektursystem zu entlasten. 
#
# Hier können URLs von Servern oder auch nur einzelnen serverseitigen Skripten
# eingetragen werden, von denen bekannt ist, dass sie ausschließlich JavaScript
# ausgeben.
# Alle zutreffenden URLs werden dann grundsätzlich als JavaScript behandelt und
# reduzieren den Verarbeitungsaufwand bei der Content-Type-Korrektur.
#
# Das ist besonders hilfreich, wenn man es mit Servern zu tun hat, deren dynamisch
# erzeugte URLs eine Zufallskomponente enthalten, die Proxy-Caches überlisten soll.
# In diesem Fall würden sich schnell Hunderte von Einträgen in der virtuellen Liste
# ansammeln.
# Ein mit geeigneten Platzhaltern formulierter statischer Eintrag löst das Problem sofort.
#
#
#
# Stand: 19.02.2010
#
# Michael Bürschgens
# http://www.buerschgens.de/Prox/
#
