# CorrectContentTypes (NoAddURL)
#
# Diese Liste wird vom Header-Filter "Content-Type: (In) Falsche Angaben regelbasiert korrigieren [MB]"
# genutzt.
#
#
# Jede Zeile ist ein unabhängiger Ausdruck.
#
# Geprüft wird grundsätzlich die Dateiendung der zu übertragenden Datei.
# Zusätzlich kann mit $TST(\1=...) der vom Server gesendete Content-Type geprüft werden.
# Mit $SET(\0=...) wird der Content-Type festgelegt, der vom Filter gesetzt werden soll.
# Wenn der vorhandene Content-Type beibehalten werden soll, kann mit $SET(\0=\1) der alte Wert übernommen werden.
#
# Außerdem kann mit $FILTER(true) die Filterung einer Übertragung aktivert werden,
# obwohl der Content-Type normalerweise eine Filterung verhindern würde.
# Dabei ist zu beachten, dass Filter, die z.B. mit $TYPE(htm) auf einen bestimmten Dateityp festgelegt sind,
# trotz manuell aktivierter Filterung nur "ihren" Dateityp filtern.
# Im Zweifelsfall bietet sich deshalb die Änderung des Content-Types nach "text/html" an.
#
# Wenn ein Content-Type unabhängig von der Dateiendung geändert werden soll,
# dann kann anstelle der Dateiendung "\w" geschrieben werden.
# In diesem Fall MUSS aber eine andere Abfrage vorhanden sein, damit die Regel nicht
# für jede Übertragung wahr wird.
#
# Auch die Unterbrechung einer bestimmten Übertragung ist hier möglich.
# Die Regel muss dafür nur den Befehl $SET(CTKill=true) enthalten.
#
# 
# 
# Stand: 16.08.2009
# 
# Michael Bürschgens
# http://www.buerschgens.de/Prox/




##### Bestimmte, eindeutig korrigierbare, falsche Content-Type-Angaben #####

\w   $TST(\1=text/x-js|application/javascript|text/script|text/x-javascript) $SET(\0=text/javascript)


##### Komplexere Ersetzungen, bei denen es auf die Kombination #####
##### aus Dateiendung und Content-Type ankommt.                #####

# Falls der Server über eine IP-Adresse angesprochen wird, dann alle Übertragungen mit
# dem Content-Type "text/plain" unterbrechen.
# \w $URL(*([0-9]+{1,3}.)++{3}[0-9]+{1,3}*) $TST(\1=text/plain) $SET(CTKill=true)
